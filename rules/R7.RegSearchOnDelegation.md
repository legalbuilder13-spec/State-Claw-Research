# R7.RegSearchOnDelegation

**Spec:** When any retrieved statutory section contains a delegation clause — language directing an agency to promulgate, adopt, or enforce rules (e.g., "the [agency] shall promulgate regulations to implement this section," "rules adopted by the [board] pursuant to this chapter," "as provided by the [department's] regulations") — the orchestrator must add an implementing-regulation search step to the plan and execute it before the analysis closes.
**Enforcement point:** The orchestrator's plan validator runs after every Statute sub-agent dispatch; it scans the returned statutory text for delegation clauses (regex pattern catalog + LLM-assisted detection for ambiguous cases) and amends the plan to include a `regulation-research` step for the named agency.
**Severity:** blocking.
**Failure mode:** If the plan is missing a regulation-search step for a delegation clause that was detected in retrieved statutory text, the orchestrator amends the plan and re-dispatches. If after the regulation search the implementing reg is genuinely not found (e.g., the agency has not yet promulgated, or the reg is in a non-allowlisted source), R15 CompletenessGate fires with criterion "regulations" failing and the deliverable is blocked or annotated as "incomplete: implementing regulation not located."

## Detail

R7 catches one of the most common errors in statutory research: stopping at the statute when the statute itself directs the reader to consult a regulation. The operative law is often in the regulation — the statute creates the framework, the agency fills in the operational details. An answer about California's healthcare-staffing requirements that cites Lab. Code without checking the relevant CCR sections is incomplete by definition.

**Delegation-clause detection.** The orchestrator runs a two-pass detection on every statutory section returned by the Statute sub-agent:

1. **Regex pass.** A catalog of delegation patterns (`shall promulgate`, `shall adopt rules`, `rules pursuant to`, `as the [\w]+ may prescribe`, `regulations [\w]+ shall [\w]+`, etc.) matched against the cleaned section text. Catalog lives at `extensions/agentic-law-os/r7-config.json`. False positive rate is acceptably low (~5% per internal benchmark) but recall is high.
2. **LLM pass.** Regex-flagged sections are inspected by a small classifier sub-agent that answers a binary question: "Does this section delegate to an agency to promulgate regulations? If yes, name the agency." This filters regex false positives and extracts the agency name for the follow-on search.

Sections that survive both passes are added to a `pending_delegations` list on the TaskSpec. The orchestrator's plan validator refuses to close the analysis until each entry has a corresponding regulation-research step that either (a) located the implementing reg, or (b) confirmed no reg exists in the allowlisted sources (and flagged the gap).

**What "implementing regulation" means.** The regulation must implement *this specific delegation*, not just be from the same agency. A statute delegating to Cal-OSHA to promulgate workplace safety rules is implemented by the Cal-OSHA-promulgated CCR sections, not by other Labor Code sections or by Cal-OSHA's general policies. The regulation-research step (handled by the Regulation sub-agent loading `skills/regulation-research/SKILL.md`) is responsible for this targeted match.

**Agency disambiguation.** When the statute names the agency abstractly ("the Director of the Department of Industrial Relations"), the orchestrator maps to the concrete agency identifier from `company-context/agency-registry.yaml` (Phase 1 deliverable) before issuing the regulation search. This decouples the statute's possibly-outdated agency name from the current operational agency code in the regulation database.

**No reg found.** Not every delegation has resulted in promulgated regulations — some delegations are dormant, some are recent and the agency hasn't acted yet, and some are implemented in informal guidance rather than formal rules. When the regulation search returns nothing, R7 doesn't block delivery outright; instead, the gap is recorded on the deliverable as `delegation_unresolved: [{ statute: "...", agency: "...", reason: "no implementing regulation located in <profile> allowlist" }]`. R15 then surfaces this in the completeness section, and the human reviewer sees the gap explicitly rather than the agent silently moving past it.

## Interactions with other rules

- **R8 CrossRefTrace** is the section-to-section cousin: R8 traces explicit cross-references within the statute (`as defined in section X`); R7 traces statute-to-regulation delegations.
- **R9 DefinitionsFirst** is a related plan-validator rule: like R7, it adds a step to the plan when the orchestrator detects a missing dependency. Both run at `before_agent_run`.
- **R15 CompletenessGate** consumes R7's status: an unresolved delegation flagged by R7 is one of the 7 criteria R15 checks. If R7 found a delegation and the regulation search failed, R15 records "regulations: incomplete" and the deliverable is annotated accordingly.

## Test cases

✅ Compliant: delegation detected and reg retrieved
```
Statute sub-agent retrieves Cal. Lab. Code § 2810.5 which contains: "The Labor Commissioner shall adopt regulations specifying the form of the notice required by this section."
R7 detection (regex + LLM): yes, delegation to Labor Commissioner.
Orchestrator adds regulation-research step: "search Cal. Code Regs. tit. 8 for sections implementing § 2810.5".
Regulation sub-agent retrieves Cal. Code Regs. tit. 8 § 11760 (the form-of-notice regulation).
R15 sees: { delegations: [{ statute: "ca-lab-2810.5", reg: "8-ccr-11760", status: "resolved" }] }
Result: deliverable includes both the statute and the implementing regulation in the analysis.
```

❌ Non-compliant: delegation detected but plan missing reg-search step
```
Statute sub-agent retrieves § 2810.5.
R7 detection: yes, delegation.
Plan as authored by Drafter: does NOT include a regulation-research step.
Orchestrator plan validator: { block: true, reason: "R7: plan does not include regulation-research for detected delegation in § 2810.5 (Labor Commissioner). Amending plan to include reg-research step." }
The orchestrator amends and re-dispatches; the next iteration includes the reg search.
```

❌ Non-compliant: delegation present, reg search returned nothing, deliverable closed anyway
```
Statute: § 2810.5 (delegation present).
Reg search: returns NO_MATCH (assume the relevant CCR section isn't yet ingested for some reason).
Drafter attempts to close the analysis without surfacing this.
R15 fires: { block: true, reason: "R15: completeness criterion 'regulations' failed — delegation in § 2810.5 has no resolved implementing regulation. Either complete the reg search (re-ingest Cal. Code Regs. tit. 8) or deliver with explicit \"delegation_unresolved\" annotation." }
```

✅ Compliant: delegation absent
```
Statute: § 226 (wage statement requirements). No delegation language.
R7 detection: no delegations.
Plan proceeds without a regulation-research step (or with one only if a separate trigger requires it).
Result: deliverable closes when other rules' criteria are satisfied.
```

⚠️ Edge case: ambiguous delegation
```
Statute: § X contains "the Director may, in his discretion, provide further guidance."
Regex pass: matches "may ... provide further guidance" pattern.
LLM pass: classifies as discretionary, NOT a true delegation (no mandatory rulemaking).
Orchestrator does NOT add a reg-search step. The discretionary-guidance pattern is logged for review but doesn't gate.
```

## Open questions

- **Regex-vs-LLM cost.** The LLM disambiguation pass costs ~1 small-model call per regex match. For a deliverable that retrieves 10 statutory sections with ~30% regex hit rate, that's ~3 LLM calls. Affordable but should be monitored — if delegation detection becomes a hot path, consider replacing the LLM with a fine-tuned classifier.
- **Agency-name registry currency.** `company-context/agency-registry.yaml` maps statutory agency references to current operational agency codes. This drifts over time as agencies are renamed or restructured. Phase 1 deliverable. Schedule annual review.
- **Multi-step delegations.** A statute may delegate to agency A which has delegated to agency B which has actually promulgated the rules. R7's current implementation traces only one hop; multi-hop delegation chains will produce false negatives. Acceptable for v1; revisit if multi-hop patterns become common (likely in federal-state delegated programs like Medicaid).
- **Informal guidance.** Some agencies issue interpretive guidance (FAQs, opinion letters) that has weight short of formal regulations. These are orientation-only (R14) and don't satisfy R7's "implementing regulation" requirement. The deliverable can still mention them as orientation, but R7's gate is for promulgated regs only. This is intentional.
