# R3.NoMemoryStatutes

**Spec:** Every statutory or regulatory assertion in any draft, claim, or sub-agent artifact must trace to at least one entry in `retrieval_log` retrieved this session. Assertions about statutory or regulatory text drawn from the model's training memory — without a corresponding retrieval — are stripped from the draft and flagged.
**Enforcement point:** Drafter output schema (every claim must carry at least one `cites[]` entry whose `doc_id` appears in `retrieval_log`); plus the Critic sub-agent which independently scans the draft for unanchored statutory assertions and triggers a `before_agent_finalize` revise loop.
**Severity:** blocking.
**Failure mode:** The unanchored sentence is stripped from the draft and the Drafter is invoked again with a `revise` action whose instructions list every stripped sentence and the rule violation. If the Drafter persistently emits unanchored statutory assertions after 2 revise loops, R11 LowConfidenceGate fires and the deliverable is blocked with "research incomplete — Drafter unable to produce a grounded answer."

## Detail

R3 is the broader umbrella over R4. Where R4 polices *quoted* text (the strict subset that requires character-level anchors), R3 polices *every* statutory or regulatory assertion in the draft — whether quoted, paraphrased, or summarized. The principle is the same: the model cannot speak about law it didn't retrieve.

**Why this rule exists.** LLMs trained on legal corpora have memorized large chunks of statutes and case law. When asked about California's AB5, the model can produce convincing-sounding statements about the ABC test from memory alone — and frequently those statements are wrong in subtle ways (the 2019 version of the test, not the post-amendment 2024 version; a paraphrase that drops a load-bearing qualifier; a citation to a section number that's correct in spirit but wrong by one). R3 makes this failure mode mechanically impossible: if the model didn't retrieve the section this session, it can't write about that section in the draft.

**Mechanism.** Every claim in the Drafter's output is required by the schema to include a `cites[]` array of at least length 1, each with a `doc_id` that exists in the session's `retrieval_log` (verified at schema-validation time by a custom AJV keyword `existsInRetrievalLog` registered on the Drafter's validator). Pure transitional sentences ("In summary…", "Turning to the second issue…") are exempt because they're explicitly typed as `kind: "transition"` claims; the schema only requires citations on `kind: "assertion"` claims.

**Paraphrase vs. quote.** R3 catches an assertion like *"California's ABC test requires that the worker be free from the hiring entity's control"* — that's a statutory paraphrase. If the Drafter has retrieved Lab. § 2775(b) this session, this passes (the claim has a cite to ca-lab-2775, and the paraphrase fidelity is separately checked by the Critic and W2 ParaphraseDriftCheck). If the Drafter has NOT retrieved 2775, R3 strips the sentence — even though the model "knows" the right answer. The fix is to retrieve, then re-Draft.

**The Critic re-check.** Schema validation runs at the Drafter's output gate. The Critic adds a second layer: it scans the rendered draft for statutory-sounding phrases (regex patterns for code-style citations, "the statute requires," "the regulation states," etc.) and verifies each maps to a `cites[]` entry. Phrases that match the statutory-sounding pattern but have no corresponding cite are flagged. The Critic's flags are evaluated by the orchestrator; persistent violations escalate to R11.

**Negative scope.** R3 governs *statutory and regulatory* assertions. Case-law summaries are governed by their own rule (R5 HashEcho + R6 PinpointCite for the cite shape, with the Verifier confirming the holding matches the cited passage). General legal principles ("burden of proof," "summary judgment standard") that aren't tied to a specific statute or case are not in R3's scope — the agent treats those as orientation.

## Interactions with other rules

- **R4 QuotingRule** is the strict subset for *quoted* text. R3 catches paraphrased assertions; R4 catches quoted ones.
- **R5 HashEcho** is the Verifier-side check for anchored spans. R3 prevents unanchored assertions from being written in the first place; R5 catches forged anchors after the fact.
- **R6 PinpointCite** is about citation *form* (title + section + subdivision). R3 is about citation *existence*. Both must pass.
- **R8 CrossRefTrace** ensures that every cross-reference in a retrieved section was itself retrieved. R3 leverages this: an assertion that depends on a cross-referenced section fails R3 unless the referenced section is also in `retrieval_log`.
- **R11 LowConfidenceGate** is the escalation path when R3 fails repeatedly.
- **W2 ParaphraseDriftCheck** is the cousin warning that fires when a paraphrase drifts between revise loops, even if both versions pass R3's existence check.

## Test cases

✅ Compliant: paraphrase backed by retrieval
```
retrieval_log contains: { doc_id: "ca-lab-2775", text: "<full text of CA Lab Code § 2775>", hash: "sha256:..." }
Drafter emits claim:
{ "kind": "assertion", "text": "California Labor Code § 2775 codifies the ABC test, requiring the hiring entity to satisfy three prongs to establish a worker is an independent contractor.", "cites": [{ "kind": "primary_statute", "doc_id": "ca-lab-2775", "section": "2775" }] }
Result: schema validation passes (cite's doc_id is in retrieval_log); Critic confirms; accepted.
```

❌ Non-compliant: assertion with no retrieval backing
```
retrieval_log is empty (or contains only unrelated entries).
Drafter emits:
{ "kind": "assertion", "text": "California's ABC test was codified in Labor Code section 2775 in 2019.", "cites": [] }
Result: Drafter schema rejects — assertion claims require non-empty cites[]. { revise: true, instructions: "R3: assertion lacks retrieval-backed citation. Either retrieve the referenced section before asserting, or remove the assertion." }
```

❌ Non-compliant: cite to doc_id not in retrieval_log
```
retrieval_log contains only ca-lab-2776 (a different section).
Drafter emits:
{ "kind": "assertion", "text": "Section 2775 codifies the ABC test.", "cites": [{ "kind": "primary_statute", "doc_id": "ca-lab-2775", "section": "2775" }] }
Result: schema rejects — doc_id "ca-lab-2775" is not in retrieval_log this session. { revise: true, instructions: "R3: cited doc_id ca-lab-2775 not retrieved this session. Retrieve via kb_statutes_get before citing, or cite the section you actually retrieved (ca-lab-2776)." }
```

❌ Non-compliant: paraphrase with subtle drift (caught by Critic, not schema)
```
retrieval_log contains: ca-lab-2775 with text "a person providing labor or services for remuneration shall be considered an employee"
Drafter emits:
{ "kind": "assertion", "text": "Section 2775 establishes that workers paid by piece-rate are employees by default.", "cites": [{ "doc_id": "ca-lab-2775" }] }
Result: schema passes (cite exists). Critic flags: the paraphrase introduces "piece-rate" which isn't in the cited section. { revise: true, instructions: "R3 Critic: paraphrase of ca-lab-2775 introduces a qualifier (\"piece-rate\") not present in the source. Retrieve the actual provision governing piece-rate workers (if any) or remove the qualifier." }
```

✅ Compliant: transition sentence (kind: transition)
```
Drafter emits:
{ "kind": "transition", "text": "Turning to the second issue, the implementing regulation governs daily-rate workers.", "cites": [] }
Result: schema passes — transition claims do not require cites. (The next assertion in the draft must, though.)
```

❌ Non-compliant: orientation source as the sole cite
```
retrieval_log contains: somelawfirm-blog-ab5 (orientation-only)
Drafter emits:
{ "kind": "assertion", "text": "AB5 codified the ABC test in California.", "cites": [{ "kind": "law_firm_blog", "doc_id": "somelawfirm-blog-ab5" }] }
Result: R14 fires (orientation_only source in cites). Even though R3's existence check passes, R14 strips the cite, leaving the assertion uncited, which then fails R3. The Drafter is told to either retrieve a primary source or remove the assertion.
```

## Open questions

- **Statutory-sounding phrase patterns.** The Critic's regex catalog for detecting unanchored statutory-sounding sentences is a configuration file at `extensions/agentic-law-os/critic-config.json`. The patterns are conservative (better to false-positive and let the Drafter re-draft than to miss a violation). Tuning these patterns is a Phase 4 task with real escalation transcripts as the corpus.
- **General legal principles boundary.** Where does "general legal principle" end and "statutory assertion" begin? E.g., "the IRS uses common-law agency factors" — is that a statutory assertion (about 26 USC) or a general principle? Current rule: any statement that names a specific statute, section, or regulation is in R3 scope; statements about general doctrines are out of scope. The Critic operates on this heuristic; manual review will refine.
- **Multi-session memory.** A future enhancement allows the agent to reference research from a prior session ("In the March memo, we established X"). When that lands, R3 must extend to permit retrievals from *prior-session* retrieval_logs that are explicitly attached to the current TaskSpec. Not in v1 scope.
