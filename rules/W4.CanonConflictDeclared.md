# W4.CanonConflictDeclared

**Spec:** When the Drafter applies more than one canon of construction (from the catalog in `skills/canon-application/SKILL.md`) to the same statutory question and the canons would produce different outcomes, the Drafter must explicitly declare the conflict in a structured `canon_conflicts[]` block: which canons were considered, what each canon's outcome would be, which canon was applied, and why. Undeclared conflicts trigger this warning.
**Enforcement point:** The Critic sub-agent scans the Drafter's claim ledger for evidence of multiple canon applications to the same statutory question; if two or more canons fire on the same claim AND the claim has no `canon_conflicts[]` block resolving them, the Critic adds a warning.
**Severity:** warning.
**Failure mode:** Deliverable is annotated. The reviewer sees: "⚠️ Canon conflict not declared. The Drafter applied both *expressio unius est exclusio alterius* and *ejusdem generis* to § 2775(b) and reached the conclusion that the exemption applies, but did not declare which canon won or why. Reviewer: verify the chosen canon and the basis for choosing it."

## Detail

W4 catches a subtle reasoning failure that's common in LLM-generated legal analysis: the model applies several canons of construction implicitly during analysis, but doesn't surface the choice between them in the output. Two canons can produce opposite results — *expressio unius* (the expression of one thing excludes others) cuts against an implied exemption; *ejusdem generis* (a general term following specific terms takes meaning from the specifics) might support one. When the Drafter chooses one canon over the other without saying so, the reviewer can't audit the choice — and a wrongly-chosen canon can flip the conclusion.

The PRD §6 Methodology Part IV ("Interpretation framework — operator words, ambiguity ladder, 17 canons of construction") makes canon application a first-class step of statutory analysis. W4 ensures the canon-application step is auditable when it produces conflicts.

**The 17 canons.** Per PRD §6 Part IV, the agent applies canons from a catalog. The full list lives in `skills/canon-application/SKILL.md` (Phase 1 deliverable). Common canons that cause conflicts:

- *Expressio unius est exclusio alterius* — the explicit listing of items excludes unlisted ones.
- *Ejusdem generis* — a general term following specific terms is read in light of the specifics.
- *Noscitur a sociis* — a word is known by the company it keeps.
- *Plain meaning rule* — the statute's text controls unless absurd.
- *Constitutional avoidance* — interpret a statute to avoid constitutional questions if possible.
- *Rule against surplusage* — interpret to give every word effect.
- *Remedial-statute liberal construction* — remedial statutes are construed liberally to effect their purpose.
- *Statute-in-derogation-of-common-law strict construction* — statutes in derogation of common law are construed strictly.
- *In pari materia* — read related statutes together.
- ... (and others; full catalog in the skill file)

Many of these can be in tension. When the Drafter's reasoning trail shows two or more canons fired on the same question, the conflict must be declared.

**Conflict-declaration shape.** The `canon_conflicts[]` block:

```json
[
  {
    "claim_id": "claim-007",
    "question": "Does subdivision (b)'s exemption apply to a worker meeting subdivision (a)'s presumption?",
    "canons_considered": [
      { "canon": "expressio_unius_est_exclusio_alterius", "outcome": "exemption does NOT apply because (b)'s list does not include this worker's situation" },
      { "canon": "ejusdem_generis", "outcome": "exemption MAY apply because the unlisted situation is of the same kind as the listed ones" }
    ],
    "applied": "expressio_unius_est_exclusio_alterius",
    "rationale": "The statute's structure of an exhaustive numbered list at (b)(1) through (b)(8) signals legislative intent to enumerate exhaustively rather than illustratively; expressio unius is the better fit. CA appellate court decisions on parallel exemption structures in the Lab. Code support this choice (cite Soto v. Castlerock).",
    "cite": "..."
  }
]
```

The structure forces the Drafter to name both canons, both outcomes, and the rationale for the choice. The reviewer can then audit.

**Detection mechanism.** The Critic compares the Drafter's claim text against the canon catalog using a pattern-matching approach (each canon has signal phrases the Drafter would use: "the listing in (b) is exclusive" → *expressio unius*; "(b)(8) reads in light of (b)(1)-(7)" → *ejusdem generis*). When two or more signal phrases fire on the same claim AND no `canon_conflicts[]` entry resolves them, W4 fires.

**Why this is a warning, not blocking.** Sometimes a canon conflict is so well-resolved by the surrounding analysis that an explicit declaration would be redundant. A warning rather than a block lets the reviewer make this judgment.

**Why this rule exists at all.** Without W4, a Drafter could quietly pick the canon that supports the desired conclusion. With W4, the choice is surfaced for audit. This doesn't prevent wrong canon-choice, but it makes wrong choices auditable rather than invisible.

## Interactions with other rules

- **W3 HedgeDensity** can co-fire when a Drafter hedges around a canon conflict ("the better view is...") instead of declaring it. The combination of W3 + W4 is a strong signal that the analysis is dodging a real interpretive choice.
- **R11 LowConfidenceGate** can fire if multiple undeclared canon conflicts indicate the analysis is unresolved at a deeper level.
- **R6 PinpointCite** is required for any case-law cite supporting a canon choice; W4 doesn't replace R6.

## Test cases

✅ No warning: declared canon conflict
```
Drafter emits canon_conflicts[]:
[
  {
    claim_id: "claim-007",
    question: "Does subdivision (b)'s exemption apply?",
    canons_considered: [expressio_unius, ejusdem_generis],
    applied: "expressio_unius",
    rationale: "Subdivision (b) is structured as an exhaustive numbered list; CA appellate authority on parallel statutes treats such lists as exhaustive.",
    cite: "<case citation>"
  }
]
W4 does not fire. Reviewer can read the declared conflict and the canon-choice rationale.
```

⚠️ Warning fires: implicit canon conflict
```
Drafter's claim text reads: "Subdivision (b)'s list of exemptions is exhaustive, so a worker not within the list is subject to the (a) presumption."
Critic detects: signal phrase "list of exemptions is exhaustive" → expressio unius applied.
Critic also detects: an adjacent paragraph reads "Even if the list is read with surrounding (b)(7)'s catch-all in mind..." → ejusdem generis or noscitur a sociis was at least considered.
canon_conflicts[]: empty.
W4 fires. Annotation: "⚠️ Canon conflict not declared. The Drafter applied expressio unius est exclusio alterius but a parallel paragraph considered ejusdem generis or noscitur a sociis. Either declare the conflict in canon_conflicts[] or remove the alternative-canon language."
```

⚠️ Warning fires: hedging instead of declaration
```
Drafter writes: "The better view is that subdivision (b) is exhaustive, though some commentators have argued for a more flexible reading."
Critic detects: hedge phrase "the better view is" → implicit canon disagreement.
W4 fires alongside W3 (hedge density). Reviewer sees both annotations: "⚠️ Canon conflict not declared (W4). ⚠️ Hedge density (W3)."
```

✅ No warning: single canon applied, no conflict
```
Drafter's claim: "Section 2775(a)'s plain meaning makes the worker an employee."
Only the plain-meaning rule was applied. No conflict.
W4 does not fire.
```

⚠️ Warning fires: surplusage rule conflict with plain meaning
```
Claim: "The phrase 'and direction' in the statute is redundant of 'control' and adds nothing."
This implicitly rejects the rule-against-surplusage canon. The Critic flags this as a canon position taken without declaration.
W4 fires. Annotation: "⚠️ The claim treats 'and direction' as redundant of 'control', implicitly rejecting the rule against surplusage. Declare why the rule against surplusage does not apply here (e.g., the legislature used both terms because of practice in other Code sections) or revise to give 'and direction' independent effect."
```

## Open questions

- **Canon-signal-phrase catalog.** The Critic's detection of which canons fired depends on a catalog of signal phrases per canon. The catalog needs careful authoring (some canons have many phrasings; some are easy to confuse). Phase 1 deliverable as part of `skills/canon-application/SKILL.md`.
- **Per-jurisdiction canon weight.** Some canons have stronger force in some jurisdictions (e.g., the *Chevron* deference doctrine post-2024 changes at the federal level; the *expressio unius* canon is treated cautiously in CA per Soto). The Drafter's rationale should account for jurisdictional weight; W4 doesn't currently enforce this but the skill file should.
- **Critic confidence on canon detection.** The Critic's signal-phrase detection is heuristic and produces both false positives and false negatives. Calibrate against production deliverables in Phase 4.
- **Catalog size.** PRD §6 references 17 canons. Some legal traditions list 100+ canons. v1 catalog is the 17 in the PRD; expansions handled in Phase 4 as needed.
