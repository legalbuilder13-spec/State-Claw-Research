# R8.CrossRefTrace

**Spec:** Every cross-reference appearing in a retrieved statutory or regulatory section — including section references (`as defined in section X`, `subject to the requirements of subdivision (Y)`), code references (`Chapter Z of Title W`), and external references (`pursuant to Section 12940 of the Government Code`) — must be traced one hop: the referenced provision must itself be retrieved before the analysis using the parent section closes.
**Enforcement point:** The Statute and Regulation sub-agents run a post-process pass on every retrieved section, extract all cross-references (via regex catalog + LLM disambiguation for ambiguous phrasings), and emit a `pending_cross_refs[]` list. The orchestrator's plan validator refuses to close until every entry is either retrieved or explicitly skipped with a recorded reason.
**Severity:** blocking.
**Failure mode:** If the analysis attempts to close with unresolved cross-references, the orchestrator amends the plan to add the missing retrievals and re-dispatches the affected sub-agent. After 3 trace-and-retrieve iterations without resolution (a closed-loop check to prevent infinite cross-ref following), R15 CompletenessGate fires with criterion "cross-refs" failing and the deliverable is blocked or delivered with the missing references explicitly annotated as gaps.

## Detail

R8 prevents one of the most insidious legal-research failure modes: the agent reads a section that says "the worker is an employee unless the employer can establish the three conditions of subdivision (b)" — and then concludes the worker is an employee, without ever retrieving subdivision (b) to check the conditions. The operative legal content is in the cross-reference, and skipping it produces a wrong answer that *looks* correct because the surface citation is to a real section.

**Cross-reference taxonomy.** R8 catches three flavors:

1. **Internal section references** within the same chapter or title (e.g., "subject to subdivision (b)" or "as provided in section 2776"). Resolved by retrieving the named section from the same source.
2. **External code references** to other titles or codes (e.g., "Section 12940 of the Government Code"). Resolved by retrieving from the named code, which may require a separate corpus query (Government Code vs. Labor Code).
3. **Cross-jurisdictional references** (e.g., a state statute that references a federal regulation). Resolved by retrieving from the federal corpus; R2 JurisdictionLock requires "US" to be in TaskSpec.jurisdictions or the cross-ref is flagged as out-of-scope.

**Detection.** The Statute sub-agent's post-process scans the retrieved section text with a regex catalog (`extensions/agentic-law-os/r8-config.json`) covering common cross-ref phrasings, then runs an LLM disambiguation pass on each match to filter false positives (e.g., "section 12940" inside a quote from a different section, or `§` used as a non-section marker). Output is a structured list:

```json
{
  "parent": { "doc_id": "ca-lab-2775", "section": "2775" },
  "pending_cross_refs": [
    { "kind": "internal_section", "target_section": "2776", "context": "..." },
    { "kind": "external_code", "target_code": "Gov.", "target_section": "12940", "context": "..." }
  ]
}
```

**One-hop only.** R8 traces one hop. If section X references section Y, R8 requires Y to be retrieved. If Y itself references section Z, R8 fires on the retrieval of Y, and Z must be retrieved. This means a long cross-reference chain is followed step by step — each hop is a separate retrieve-and-analyze step, and at each step the agent can stop if the chain is no longer relevant to the original question.

The alternative (depth-first trace until exhaustion) was rejected for two reasons: (a) it produces unbounded retrieval costs for deeply-cross-referenced statutes; (b) it produces noise that drowns the actual question. One-hop with explicit chaining at the orchestrator level keeps the agent's behavior predictable.

**Explicit skip.** Not every cross-reference is relevant to the question at hand. The Drafter can mark a cross-ref as `skipped: { reason: "..." }` (e.g., "subdivision (b) addresses appellate procedure and is not relevant to the substantive question") and R8 accepts the skip. Skips are surfaced in the deliverable's `cross_refs` block so a reviewer can audit the agent's relevance judgments.

**Loop protection.** Cross-references can form cycles (section A references section B which references section A). R8 caps the trace depth at 3 hops per chain and 10 total cross-ref retrievals per deliverable; beyond that, the deliverable is annotated with "cross_ref_limit_reached" and a reviewer is asked to deepen manually.

## Interactions with other rules

- **R7 RegSearchOnDelegation** is the statute-to-regulation case; R8 is the statute-to-statute (or reg-to-reg) case. Both run at the same plan-validation gate.
- **R9 DefinitionsFirst** is a special-case predecessor of R8: the Definitions section reference is always the first cross-ref to trace, before any operative-provision cross-ref.
- **R15 CompletenessGate** treats unresolved cross-refs as a completeness failure (criterion 5 of 7).
- **W1 PartialAcrossStates** can fire if a cross-ref points to another state's law and that state isn't in TaskSpec.jurisdictions — the cross-ref is "skipped" with a reason, and the deliverable surfaces the gap.

## Test cases

✅ Compliant: internal cross-ref traced
```
Statute sub-agent retrieves Cal. Lab. Code § 2775(a) which contains: "Notwithstanding any other provision of law, for purposes of the provisions of this code and the Unemployment Insurance Code, and for the wage orders of the Industrial Welfare Commission, a person providing labor or services for remuneration shall be considered an employee rather than an independent contractor unless the hiring entity demonstrates that all of the following conditions are satisfied: (1) The person is free from the control and direction of the hiring entity in connection with the performance of the work..."

R8 detection: cross-refs to "the wage orders of the Industrial Welfare Commission" and "the Unemployment Insurance Code".
Orchestrator adds retrievals; both retrieved; analysis proceeds.
Result: deliverable's analysis is grounded in the parent section + both cross-referenced provisions.
```

❌ Non-compliant: cross-ref not traced
```
Statute sub-agent retrieves § 2775 with cross-ref to "subdivision (b)".
Plan does not include retrieval of subdivision (b).
Orchestrator plan validator: { block: true, reason: "R8: cross-ref to subdivision (b) of ca-lab-2775 not retrieved. Amending plan." }
After amendment: retrieval succeeds; analysis proceeds.
```

⚠️ Conditional: cross-ref to out-of-jurisdiction
```
TaskSpec.jurisdictions: ["US-CA"]
Cal. Lab. Code § X cross-references 29 C.F.R. § 541 (federal).
R8 detection: cross-ref to federal regulation.
R2 JurisdictionLock check: "US" not in scope.
Resolution: cross-ref is skipped with reason "out-of-jurisdiction (TaskSpec excludes US-federal)". Deliverable's cross_refs block records the skip. W1 may fire if the federal reg materially affects the analysis; otherwise the gap is informational.
```

✅ Compliant: explicit skip
```
Statute § Y has cross-ref to § Z which addresses appellate procedure.
Question is about substantive liability.
Drafter marks: { kind: "internal_section", target_section: "Z", skipped: { reason: "subdivision (a) governs appellate procedure, not relevant to substantive liability inquiry" } }
Orchestrator accepts skip; cross_refs block records both the reference and the skip rationale; analysis proceeds.
```

❌ Non-compliant: loop without termination
```
§ A references § B; § B references § A.
Retrievals chain: A → B → A → B → A (depth=5).
R8 caps at depth 3; on the 4th hop, the chain is terminated.
Result: deliverable annotated "cross_ref_limit_reached: chain A↔B exceeded depth 3. Manual review recommended." Deliverable still delivers with the gap recorded.
```

## Open questions

- **Cross-reference regex catalog completeness.** The current catalog covers ~95% of California Labor Code cross-ref phrasings; coverage will be lower for less-tested jurisdictions. Phase 1 should populate per-state addenda to the catalog as new states are ingested.
- **Inline references vs. footnote references.** Some statutes use footnotes for cross-references rather than inline. The Docling parser preserves footnote markers, but R8's detector currently treats footnotes as inline; this may produce false positives if a footnote merely cites authority rather than cross-referencing operative law. Phase 4 refinement.
- **Trans-codification.** When a statute has been recodified (e.g., old Lab. Code § 3353 → current § 2775), cross-references may point at the old section number. R8 currently does not resolve historical references; the corpus layer must handle aliasing. Out of v1 scope; treat historical-section references as informational.
- **Three-hop depth tuning.** The 3-hop / 10-total limits are guesses. Real workload data should adjust them. Configurable in `r8-config.json`.
