# R14.NoLawFirmCitation

**Spec:** Entries in the deliverable's `cites[]` array must be drawn exclusively from the `citation_eligible` source categories: `primary_statute`, `primary_regulation`, `case_law`, `secondary_treatise`. The `orientation_only` categories (`law_firm_blog`, `news_article`, `vendor_blog`, `wikipedia`, `ncsl_summary`, `agency_guidance`, `index_only`) are type-rejected at render time. Orientation sources, if used at all, may appear only in a separate `references[]` block where the schema labels them as "orientation, not for citation."
**Enforcement point:** Renderer output schema (`schemas/deliverable.schema.json`) defines `cites[]` as a discriminated union of the citation-eligible types only. Any cite whose `kind` is in the orientation-only enum fails schema validation in the `before_agent_finalize` hook.
**Severity:** blocking.
**Failure mode:** Render fails with `CITATION_KIND_REJECTED: cite #<index> has kind "<kind>" which is orientation_only and may not appear in cites[]. Either remove the cite, replace it with a primary source covering the same point, or move it to references[] (orientation block).`

## Detail

R14 is the rule that prevents the most common form of legitimate-looking-but-unreliable citation: the law-firm-blog citation. Law-firm blogs are written by attorneys, they sound authoritative, and they often summarize the law accurately — but they are not primary sources, they are not subject to peer review or precedential weight, and they age unpredictably (the firm may publish a corrected version next week, or the post may disappear). A research deliverable that cites a law-firm blog is functionally citing the blog author's opinion as if it were law.

R14 makes this mechanically impossible by limiting `cites[]` to four source categories that have well-defined precedential or authoritative weight:

- **primary_statute** — the legislature's actual words.
- **primary_regulation** — the agency's actual words, promulgated under proper authority.
- **case_law** — judicial opinions from courts of record.
- **secondary_treatise** — peer-recognized treatises (Restatement, Wright & Miller, etc.) that carry persuasive authority in legal argument.

These four are the only kinds that can be cited. Everything else — law firm blogs, news articles, vendor blog posts, Wikipedia, NCSL summaries, agency guidance documents (FAQs, opinion letters), and index-only sources (e.g., Justia's index, Cornell LII's index) — is `orientation_only` and may not appear in cites.

**Why orientation sources still have a use.** A law-firm blog can be a fast way to orient on a topic ("what's the big picture on AB5?"). The agent can retrieve and read these (R1 SourceAllowlist controls whether they're reachable at all — current default: not in profile-b allowlist, so they're not retrievable in v1; future versions may permit retrieval with R14 still gating citation). When orientation sources are read, they appear in `references[]`, a separate output block clearly labeled "Orientation references (not citation sources)." The reviewer can see what the agent looked at for context; the reviewer cannot mistake it for authority.

**Agency guidance is configurable.** Some operators may want to treat formal agency guidance (DOL Field Operations Handbook excerpts, IRS Revenue Rulings, DLSE Opinion Letters) as citable. The PRD §11 marks `agency_guidance` as `# operator-configurable per profile`. Profile A (Lexis-enabled) may move agency guidance to `citation_eligible`; Profile B (free) defaults to `orientation_only`. R14 reads the live profile to make this determination.

**The four citation-eligible types are themselves enforced.** R6 PinpointCite requires each citation-eligible type to have its specific pinpoint fields. R14 is the kind-level gate (which kinds are allowed); R6 is the field-level gate (what each allowed kind requires).

**Why "law firm blog" gets singled out in the rule name.** Of all the orientation sources, law-firm blogs are the most dangerous because they look authoritative. News articles obviously aren't authority; Wikipedia obviously isn't authority. A law firm publishing under attorney bylines with footnotes *looks* like authority. The rule name surfaces this risk explicitly.

**Audit chain logs both cites and references.** Every cite in `cites[]` is logged. Every entry in `references[]` is also logged separately so reviewers can audit what orientation material was consulted. This separation is what makes "I looked at it for context but didn't cite it" auditable rather than implicit.

## Interactions with other rules

- **R1 SourceAllowlist** controls whether an orientation source can be fetched at all (per the source profile). R14 controls whether any retrieved source can be cited. R1 fires upstream; R14 fires downstream.
- **R3 NoMemoryStatutes** requires every assertion to trace to retrieved content; if the Drafter retrieves only orientation sources and tries to cite them, R3 catches the existence-but-R14 catches the kind. Both gates fire.
- **R6 PinpointCite** is the field-level gate within the citation-eligible kinds; R14 is the kind gate. Both must pass.
- **R12 ClassificationOverlay** is unrelated but adjacent — both run at render time. R12 is about the company-context overlay; R14 is about citation kinds.

## Test cases

✅ Compliant: all cites are citation-eligible
```
cites: [
  { kind: "primary_statute", ... },
  { kind: "case_law", ... },
  { kind: "primary_regulation", ... }
]
references: [
  { kind: "law_firm_blog", url: "https://...", title: "...", consulted_for: "general orientation on AB5 history" }
]
Schema passes. Render proceeds. The Slack response includes a "Sources:" section listing only the cites, and a separate small "Orientation consulted:" block listing the law-firm blog (clearly labeled non-authority).
```

❌ Non-compliant: law-firm-blog cite
```
cites: [
  { kind: "law_firm_blog", url: "https://somelawfirm.com/ab5-explained", title: "Understanding AB5" }
]
Schema rejects: { revise: true, instructions: "R14: CITATION_KIND_REJECTED: cite #1 has kind \"law_firm_blog\" which is orientation_only and may not appear in cites[]. Either remove the cite, replace it with a primary source covering the same point, or move it to references[]." }
```

❌ Non-compliant: NCSL summary cite
```
cites: [
  { kind: "ncsl_summary", url: "https://www.ncsl.org/...", title: "State Independent Contractor Laws 2024" }
]
Schema rejects: NCSL summaries are orientation_only.
Result: Drafter consolidates the substantive points NCSL made into primary citations (the underlying statutes NCSL was summarizing) and moves the NCSL link to references.
```

⚠️ Conditional: agency_guidance under Profile A vs. Profile B
```
Profile B (active): agency_guidance is orientation_only.
Cite { kind: "agency_guidance", source: "DLSE Opinion Letter 2002.01.01" } → schema rejects.

Profile A (active in another deployment): agency_guidance is moved to citation_eligible in source-profiles/profile-a-lexis.yaml.
Same cite under Profile A: schema permits.
```

✅ Compliant: orientation reference (not in cites)
```
references: [
  { kind: "law_firm_blog", url: "https://...", title: "AB5 Overview", consulted_for: "topical orientation only" }
]
cites: [
  { kind: "primary_statute", code: "Lab.", section: "2775", ... }
]
Render passes. The blog appears in the Orientation section of the Slack template; primary cite appears in the Citations section.
```

❌ Non-compliant: Wikipedia in cites
```
cites: [{ kind: "wikipedia", url: "https://en.wikipedia.org/wiki/AB5_(California)" }]
Schema rejects. Drafter is asked to either remove or move to references.
```

## Open questions

- **Restatements and treatises eligibility.** `secondary_treatise` is citation-eligible, but defining what counts is non-trivial. Restatements clearly qualify; Wright & Miller clearly qualifies; CA Practice Guide (Rutter) probably qualifies for CA-state-court contexts; a Practising Law Institute course handbook probably doesn't. The `citation-style.yaml` config should include a treatise allowlist; deliverables should fail if they cite a treatise not on the list. Phase 1 deliverable.
- **Agency guidance per jurisdiction.** Some jurisdictions treat agency opinion letters as more authoritative than others. Per-jurisdiction overrides in `source-profiles/profile-{a,b}.yaml` may be needed.
- **Self-citing the system.** Should the agent ever cite itself or a prior deliverable? No, never — R14 doesn't include "prior deliverable" in citation-eligible. Prior research feeds the agent's context, not the deliverable's authority.
